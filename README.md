# Навыки для OpenClaw

Коллекция навыков для разработки и проверки навыков OpenClaw. Канонический
исходный набор хранится в `.apm/skills/`, а при установке через APM навыки
разворачиваются в рабочую `skills/`-поверхность целевого проекта. Пакет
помогает агенту учитывать требования OpenClaw к `SKILL.md`, загрузке навыков,
ограничениям видимости, Skill Workshop, безопасности и локальным материалам
навыка.

Сейчас коллекция содержит навык `openclaw-skill-development`. Он предназначен
для задач, где нужно создать новый навык OpenClaw, проверить существующий навык
или подготовить изменение так, чтобы оно опиралось на корпус знаний проекта.

## Установка

Подключите коллекцию в проект, где агент должен работать с навыками OpenClaw:

```bash
apm install github.com/mekras/openclaw-skills#master --target agent-skills
```

После установки навык доступен агенту среди проектных навыков. Первый рабочий
шаг — поручить агенту задачу с явным использованием
`openclaw-skill-development`, например проверку существующего `SKILL.md` на
совместимость с OpenClaw.

## Состав

- `.apm/skills/openclaw-skill-development/` — исходник навыка для создания, проверки и
  сопровождения навыков OpenClaw.
- `knowledge/` — корпус знаний по документации OpenClaw и связанным материалам.
- `apm.yml` и `apm.lock.yaml` — описание пакета APM и зафиксированные
  зависимости.

## Сопровождение

Для локальной работы с репозиторием установите зависимости из файла блокировки:

```bash
apm install --frozen
```

Скрипт `compile` пересобирает локальные инструкции для текущей цели разработки
в этом репозитории:

```bash
apm run compile
```

Проверить тесты коллекции и APM-упаковку:

```bash
apm run tests
apm run pack:dry-run
```

Перед содержательными изменениями навыков сверяйтесь с
`knowledge/index/statements.yml` и связанными материалами корпуса знаний.
